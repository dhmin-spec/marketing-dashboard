# 네이밍 컨벤션 정의서

> 소재·캠페인·그룹 코드의 **문법 원본 문서**. 파이프라인/대시보드의 파싱·자동분류가 이 규칙에 의존한다.
> 🔴 표시는 **사용자가 반드시 확정**해야 하는 부분 (데이터에서 유추 불가).

## 1. 소재(소재명) 문법

관찰된 패턴 (예시: `VID_플러스멤버십_겨울_A_v1`):

```
VID    _  플러스멤버십  _  겨울  _  A  _  v1
[타입]     [   테마   ]   [시즌]   [AB] [버전]
```

### 1-1. 타입 코드 (prefix) — 🔴 전체 목록 확정 필요

| 코드 | 의미(추정) | 확정? |
|------|-----------|-------|
| `VID` | 영상(Video) | ⬜ |
| `IMG` | 이미지(Image) | ⬜ |
| `CRS` | 캐러셀(Carousel)? | 🔴 확인 |
| `TXT` | 텍스트(검색 소재) | ⬜ |
| (그 외?) | 🔴 추가 코드 있으면 기입 | 🔴 |

### 1-2. 테마 — 🔴 카테고리 코드 사전 필요

- 관찰값: `플러스멤버십`, `적립혜택`, `할인쿠폰`, `특가`, `배송혜택`, `신상품`
- 🔴 이게 **고정 카테고리 목록**인가? 자유 텍스트인가? → 분류·집계 방식이 달라짐

### 1-3. 시즌 — 🔴 코드 체계 확인

- 관찰값: `겨울` → 봄/여름/가을/겨울 4종? 아니면 `2025Q1` 같은 코드도 쓰나?

### 1-4. AB 테스트 표기 — 🔴 가장 중요, 반드시 확정

- 관찰값: `_A_`, `_B_`, 그리고 `v1`/`v2` 버전 표기가 별도로 존재
- 🔴 **질문**: `A`/`B`가 **AB 테스트 그룹**인가? 아니면 단순 소재 변형(안A/안B)인가?
- 🔴 `v1`/`v2`는 소재 개정 버전인가?
- → AB 성과 비교 리포트를 자동화하려면 이 정의가 **필수**

## 2. 캠페인 코드 문법

관찰된 패턴 (예시: `GGL_CMP_01_플러스가입`):

```
GGL   _  CMP  _  01  _  플러스가입
[매체] [구분] [번호]  [   목적   ]
```

- 매체 prefix — 🔴 전체 목록: `GGL`(구글), `META`(메타), `NVR`(네이버), (그 외?)
- `CMP` = 캠페인 고정 태그? 🔴
- 번호 = 매체 내 캠페인 일련번호
- 목적 = `채널분류`/`캠페인목적` 컬럼과 대응 (플러스가입/리타겟팅/첫구매/신규유저/룩얼라이크/재구매/브랜드KW/일반KW)
