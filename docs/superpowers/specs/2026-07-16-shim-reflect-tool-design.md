# 심의 의견 자동 반영 도구 — 설계 스펙

- 작성일: 2026-07-16
- 상태: 승인됨 (구현 계획 대기)
- 유형: 신규 Streamlit 웹앱 (기존 대시보드와 별개)

## 1. 목적

심의안 엑셀 파일을 업로드하고 심의 의견을 붙여넣으면, AI(Claude)가 의견을
반영해 문안을 수정한 **새 엑셀 파일**을 다운로드받는 웹 도구. 팀원이 공유해서
사용하도록 사내 호스팅한다.

**반영 방식은 AI 자동 수정.** 단, 카피 최종 결정은 사람 검수 필수이므로
다운로드 전 화면에서 사람이 인라인 검수·확정하는 단계를 반드시 거친다.

## 2. 입력 데이터 구조 (실제 파일 기준)

대상 시트는 `검색광고 T&D`. 열 구조:

| 열 | 이름 | 역할 |
|----|------|------|
| B(2) | 보종 | 참고 |
| C(3) | NO | 문안 번호 (의견 매핑 키) |
| D(4) | 문구 및 이미지 | **수정 대상 문안** |
| E(5) | 글자수 | `=LEN(D)` 수식 (자동 재계산, 건드리지 않음) |
| F(6) | 광고위치 | 설명문구/추가제목/추가설명/서브링크 |
| H(8) | 비고 | **최대 글자수 (하드 제약)** |

관련 시트:
- `업로드용`: `A(보종) B(">") C(문안 고정값) D(=A&B&C 수식)`.
  C3(문안)은 T&D 참조가 아닌 **고정 복사본** → D열 수정 시 함께 갱신해야 함.
  행 순서 1:1 매핑 (T&D NO 1~12 ↔ 업로드용 R1~12).
- `드롭다운 수식`: 무수정.

**글자수 규칙**: `LEN` = 공백 포함, 한글 1자=1. Python `len()`과 일치.

## 3. 심의 의견 형태

번호별 의견, `/`로 구분. 예:

```
2번 문안: '원문...' → 수정 지시 (구체화 안내) / 6번 문안: 'AI맞춤보장' : 'AI맞춤보장보험'으로 수정
```

번호(NO)를 키로 매핑. 형식이 다소 흔들려도 Claude가 최대한 해석.

## 4. 아키텍처

- **Streamlit 단일 앱** (`shim_app.py`, 신규 파일)
- 라이브러리: `openpyxl`(서식 보존 엑셀 조작), `anthropic`(Claude API)
- API 키·비밀번호는 `st.secrets` (`.streamlit/secrets.toml`, gitignore)
- 진입: 공유 비밀번호 게이트

## 5. 파이프라인

1. **업로드** — .xlsx → `검색광고 T&D`에서 `NO, 문안(D), 광고위치(F), 최대글자수(H)` 추출
2. **의견 붙여넣기** — 자유 텍스트
3. **AI 수정** — Claude에 전체 문안 목록 + 의견 원문 전달 → 의견 있는 번호만 수정문 생성
4. **검수 (필수)** — 화면 표에서 사람이 인라인 편집·확정
5. **다운로드** — 원본 복제 후 D열(+업로드용 C3) 교체한 새 xlsx

## 6. AI 수정 로직

- Claude 입력: 각 문안의 `NO / 광고위치 / 최대글자수(H) / 원문` + 심의 의견 원문 전체
- 반환(JSON): `[{no, revised, reason, char_count}]` — **의견 없는 번호는 제외(원본 유지)**
- **최대 글자수(H) 하드 제약**: 프롬프트에 각 문안의 상한을 명시하고 그 안에서 수정하도록 지시.
  반환값 `len(revised) > H`이면:
  1. Claude에 "H자 이내로 더 줄여라" 1회 자동 재요청
  2. 그래도 초과면 화면에 **빨강 경고**로 남겨 사람이 처리 (**임의로 자르지 않음**)
- 창작 결과이므로 정확도 **"추정"** 등급 표기, 검수 전제

## 7. 엑셀 재생성 규칙 (원본 훼손 금지)

- 원본 워크북을 **복제**해 작업 (서식·수식·나머지 시트 보존)
- `검색광고 T&D`: 수정 행의 **D열 값만 덮어쓰기**. E(글자수) 수식 유지 → Excel 자동 재계산
- `업로드용`: 같은 번호 행의 **C3(문안)도 동일 덮어쓰기**. C4(`=A&B&C`)는 자동
- `드롭다운 수식` 등 나머지 시트 무수정
- 파일명: `원본명_심의반영_YYYYMMDD.xlsx`

## 8. UI / 화면 흐름 (Streamlit)

1. 비밀번호 게이트
2. 업로드 영역 + 문안 목록 미리보기 (번호·광고위치·원문·글자수/제한)
3. 심의 의견 텍스트박스
4. [AI 수정 실행] 버튼 → 스피너
5. 검수 표 `st.data_editor`: `번호│원문│수정문(편집)│글자수/제한│사유`.
   제한 초과 행 빨강, 편집 시 글자수 실시간 재계산
6. [엑셀 다운로드] 버튼 (검수 확정본)
7. 상단 고지: "AI 자동 수정본입니다. 발송 전 반드시 검수하세요"

## 9. 보안 / 데이터 취급

- 비밀번호 게이트 + API 키 서버 시크릿 (깃 노출 금지)
- 업로드 파일·의견·결과는 **세션 메모리에서만** 처리, 서버 디스크 미저장 (광고주 민감 자료)
- 외부(Claude API)로 나가는 것은 문안·의견 텍스트뿐 → 고지에 명시
- 호스팅: 사내 서버 또는 Streamlit 배포

## 10. 엣지케이스 / 에러 처리

- 시트명/열 구조가 다름 → "인식 못 함" 안내 + 문안/제한 열 수동 지정 폴백
- 의견 참조 번호가 표에 없음 → "매칭 실패" 별도 표시 (버림 아님)
- 의견 파싱 실패 → Claude 최대 해석, 실패 시 원문 유지 + 경고
- API 오류/타임아웃 → 재시도 버튼, 부분 성공분 유지
- 업로드용 행 수 ≠ T&D 문안 수 → 경고 후 번호 기준 매칭

## 11. 테스트 (pytest)

- 엑셀 재생성: 첨부 실제 파일로
  - D열 교체 후 E 수식 보존
  - 업로드용 C3 갱신·C4 자동 연동
  - 나머지 시트·서식 무변경
- 글자수 규칙: `len()` == 원본 `LEN` 결과 일치 (샘플 행 대조)
- 파싱: `/` 구분·번호 추출 케이스
- AI 호출부는 mock으로 분리, 로직만 테스트

## 12. 비목표 (YAGNI)

- 다중 광고주/시트 자동 판별 (일단 이 양식 우선, 폴백만 제공)
- 사용자별 계정/권한 (공유 비밀번호로 충분)
- 의견 이력·버전 관리
