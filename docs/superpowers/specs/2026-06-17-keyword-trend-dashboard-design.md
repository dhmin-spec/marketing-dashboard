# 키워드 효율 추이 대시보드 — 설계 문서

- **작성일:** 2026-06-17
- **목적:** 구글 시트의 여행자보험 키워드 광고 데이터를, 키워드별 **효율 지표 추이**를 시각적으로 비교하는 대시보드로 정리한다.

---

## 1. 개요

마케터가 매일 보는 키워드 광고 성과 데이터(구글 시트)를, 새로고침만 하면 최신 상태로 갱신되는 **단일 HTML 대시보드**로 만든다. 핵심 질문은 "어떤 키워드가 돈값을 하고, 그 효율이 시간에 따라 어떻게 변하나?"이다.

---

## 2. 데이터 소스

- **원본:** 구글 시트 (탭 `gid=0`), 일별 행 단위 데이터.
- **연결 방식:** 시트를 **웹에 게시(CSV)** → 대시보드가 클라이언트에서 해당 CSV를 `fetch`로 불러옴. 새로고침 시 시트 최신값 반영(실시간성 충분, 서버 불필요).
- **설정(1회):** 시트 → 파일 → 공유 → 웹에 게시 → 해당 탭, CSV 형식 → 발급된 링크를 대시보드 설정에 입력.
- **정확도:** 원본 수치 그대로 사용(파생 지표만 계산). **정확** 등급.

### 2.1 원본 컬럼 (19개)

| 분류축 | 퍼널/성과 |
|---|---|
| Date, Media, Device, Media_CPC, Part, campaign, adgroup, **keyword**, Product | impression, click, cost, sum_rank, connection, input, complete, conclusion, sales, 매출 |

- 퍼널 순서: `impression → click → connection → input → complete → conclusion`
- `conclusion` = 가입(전환) 완료 수, `매출` = 전환 매출액(원), `sales` = 매출의 포맷 문자열(중복 → 미사용).

### 2.2 파생 지표 (계산 정의)

| 지표 | 정의 | 비고 |
|---|---|---|
| CTR | click / impression | 클릭률 |
| CPC | cost / click | 클릭당 비용 |
| CVR | conclusion / click | 전환율 (핵심) |
| CPA | cost / conclusion | 전환당 비용 (핵심) |
| ROAS | 매출 / cost | 광고수익률 (핵심) |

- 분모 0인 경우 해당 지표는 `-`(N/A)로 표기, 차트에서는 결측 처리.
- 집계: 여러 행(같은 키워드·다른 날짜/매체/디바이스)은 **합산 후 비율 재계산**(가중평균). 즉 CTR은 Σclick/Σimpression으로 계산(단순 평균 금지).

## 3. 기능 요구사항

### 3.1 레이아웃 — "오버뷰 → 드릴다운" (확정안 A)

위에서 아래로 3개 영역:

1. **상단: 필터 바 + KPI 요약 카드**
   - 필터: **기간(날짜 범위)**, **매체(Media)**, **디바이스(Device)**.
   - KPI 카드: 선택된 필터 범위의 전체 합계 기준 ROAS · CVR · CPA · CTR · 비용 · 매출. 전기간 대비 증감(▲▼) 표기.

2. **중앙: 키워드 효율 추이 라인차트**
   - X축 = 날짜, Y축 = 선택 지표(CVR 기본, CTR/CPA/ROAS 토글).
   - **상위 N개 자동 표시 (확정안 ②):** 기본 정렬 = **전환수(conclusion) 상위** → 상위 5개 키워드를 자동으로 라인으로 그림.
   - 키워드 칩으로 라인 켜고 끄기. "상위 기준" 전환 가능(전환수/비용/매출/클릭).

3. **하단: 키워드 랭킹 테이블**
   - 컬럼: keyword, impression, click, CTR, cost, CPC, conclusion, CVR, CPA, 매출, ROAS.
   - 정렬 가능(기본 전환수 내림차순), 키워드 검색 박스.
   - 행 클릭 → 해당 키워드를 중앙 차트에 라인으로 추가/제거(드릴다운).

### 3.2 비주얼 스타일 — "라이트 · 클린 리포트" (확정안 B)

- 흰/연회색 배경, 부드러운 그림자 카드, **블루(#3b5bdb) 포인트 컬러**.
- 효율 좋음=초록, 나쁨=빨강의 보조 색상 규칙(증감·테이블 강조).
- 캡처해서 슬랙/노션에 붙이기 좋은 정돈된 톤.

## 4. 기술 구조

- **단일 정적 HTML 파일** (`dashboard.html`) — 별도 빌드/서버 없음. 더블클릭으로 열거나 Drive/정적 호스팅에 업로드.
- **차트:** ECharts (CDN 로드) — 다중 라인, 결측 처리, 툴팁/줌에 강함.
- **CSV 파싱:** PapaParse (CDN) 또는 경량 자체 파서.
- **구성 요소(역할 분리):**
  - `loadData()` — 게시된 CSV fetch → 행 배열.
  - `parseRows()` — 타입 변환(숫자/날짜), 결측·`-` 정리.
  - `aggregate(filters)` — 필터 적용 + 키워드×날짜 집계, 파생 지표 계산.
  - `renderKPIs()`, `renderTrendChart()`, `renderTable()` — 각 영역 렌더 (서로 독립, 같은 집계 결과를 입력으로 받음).
  - `state` — 현재 필터·선택 지표·선택 키워드 보관, 변경 시 위 렌더 함수 재호출.
- **설정값:** 게시된 CSV URL은 파일 상단 상수 1곳에서 관리.

---

## 5. 엣지 케이스 / 에러 처리

- CSV 로드 실패(게시 안 됨/네트워크) → 화면에 안내 메시지 + 재시도 버튼.
- 분모 0 → 지표 `-`, 차트 결측.
- 빈 결과(필터로 데이터 0건) → "해당 조건 데이터 없음" 안내.
- 키워드 수백 개 → 테이블은 페이지네이션/가상 스크롤 또는 상위 200개 + 검색.
- 숫자 포맷: 비용·매출은 천단위 콤마(₩), 비율은 % 소수 1자리.

---

## 6. 범위 밖 (YAGNI)

- 시트 쓰기/수정 (읽기 전용 대시보드).
- 매체 API 직접 연동 (시트가 단일 소스).
- 로그인/권한 관리 (게시 CSV로 대체).
- 키워드 자동 인사이트·AI 추천 (1차 범위 제외).
- 캠페인/광고그룹 드릴다운 필터 (이번엔 매체·디바이스까지만; 추후 확장 가능).

---

## 7. 검증 기준

- 게시된 CSV를 불러와 KPI·차트·테이블이 시트 수치와 일치(샘플 키워드 수기 검산).
- 필터(기간/매체/디바이스) 변경 시 세 영역이 일관되게 갱신.
- 전환수 상위 5개 키워드가 차트에 자동 표시, 칩·테이블 행 클릭으로 추가/제거 동작.
- 라이트 클린 스타일, 블루 포인트로 렌더.
