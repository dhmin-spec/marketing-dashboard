# 대시보드 — 일별 이상 감지(알림) 홈 설계

- 날짜: 2026-07-16
- 대상: `mkt_dashboard.py` (퍼포먼스 마케팅 대시보드, Streamlit)
- 목적: 퍼마팀이 **매일 아침 30초 안에 "오늘 손봐야 할 것"** 을 파악하는 랜딩 알림 화면

## 배경

기존 `mkt_dashboard.py`는 KPI + 자동 인사이트 + 3탭(갭/성과/EDA) 구조.
매일 여는 화면으로는 "이상 감지"가 가장 유용하다는 결론 → 랜딩 최상단에 **전일 대비 급변 알림**을 추가한다.
지표·네이밍 규칙은 `CLAUDE.md` / `docs/conventions/naming-convention.md`를 따른다.

## 요구사항 (확정)

- **메인 목적**: 이상 감지(알림형)
- **알림 기준**: 전일 대비 급변
- **감시 지표**: 비용, 클릭, 전환(=구매_매체)
- **임계값**: |변화율| ≥ 50%
- **집계 레벨**: 계정 전체 총합 (사이드바 필터 반영)
- **보강**: 급변 항목마다 원인 채널 Top(Δ 큰 순) 자동 표시

## 접근 (선택: A + 보강)

기존 대시보드에 **랜딩 상단 알림 배너**를 추가. 기존 KPI·인사이트·3탭은 유지.
계정 총합 알림의 약점(원인 불명)을 "원인 채널 breakdown"으로 보완.

## 화면 구조

```
타이틀/부제
─────────────────────────────
🚨 오늘의 알림  [기준일: D vs D-1]      ← 신규, 최상단
  급변 있음: 🔴 지표 ±N% (prev→D) / 원인 Top: 채널 +Δ ...
  급변 없음: ✅ 이상 없음 + 3지표 변화율 요약(⚪)
─────────────────────────────
💡 자동 인사이트 (기존)
KPI 행 (기존)
[🔀 갭] [📈 성과] [🔍 EDA] (기존)
```

사이드바에 **"비교 기준일" selectbox** 추가 (기본=최신일, 선택지=데이터 존재 날짜 전체).

## 알림 계산 로직

- **비교쌍**: D=최신(또는 선택)일, D-1=데이터가 존재하는 직전 날짜(연속일 아니어도 됨).
- **변화율**: `(D − D_prev) / D_prev × 100`, 지표별.
- **판정**: `|변화율| ≥ ALERT_THRESHOLD(50)` → 🔴 급변, 미만 → ⚪ 정상 참고.
- **필터 연동**: 사이드바 채널/캠페인 필터가 걸리면 그 범위 합계 기준.
- **원인 채널 breakdown**: 급변 지표를 채널별 D vs D_prev 차이(Δ) 계산 → |Δ| 큰 순 Top 3.
- **0 나눗셈**: D_prev=0 → "신규 발생(0→N)" 텍스트 표기(변화율 대신).

## 데이터 상태별 표시

| 상태 | 표시 |
|------|------|
| 2일+ / 급변 있음 | 🔴 급변 카드 N개 + 원인 채널 Top |
| 2일+ / 급변 없음 | ✅ 이상 없음 + 3지표 변화율 요약(⚪) |
| 1일뿐 | ℹ️ 전일 비교하려면 데이터 2일 이상 필요 |
| 필터 결과 0건 | ⚠️ 선택 조건 데이터 없음 |

## 코드 구성

### `mkt_pipeline.py` (순수 로직)
- `compute_alerts(df, date_d=None, threshold=50, metrics=("비용","클릭","구매_매체")) -> dict`
  - 반환: `{"D", "D_prev", "status": "ok|no_prev|empty", "items": [...]}`
  - item: `{"지표","변화율","d","prev","신규여부","원인_top":[(채널, Δ)]}`
  - 헬퍼 `_channel_deltas(df, metric, d, prev)`.
  - 순수 함수 → 단위 테스트·배치 재사용 가능.

### `mkt_dashboard.py` (렌더링)
- `render_alerts(alerts)`: 상태별 카드 UI(🔴/✅/ℹ️/⚠️).
- 사이드바 "비교 기준일" selectbox.
- 배치 순서: 타이틀 → `render_alerts()` → 자동 인사이트 → KPI → 탭.

### 상수 (파일 상단, KPI 확정 시 교체 지점)
```python
ALERT_THRESHOLD = 50
ALERT_METRICS = ["비용", "클릭", "구매_매체"]
CONV_METRIC = "구매_매체"   # AF 기준 원하면 구매_af
```

## 지표 정의

- "전환" = `구매_매체` (매체 리포트 기준, CLAUDE.md 지표 정의 준수). AF 기준 전환은 `CONV_METRIC` 상수 한 줄로 전환 가능.
